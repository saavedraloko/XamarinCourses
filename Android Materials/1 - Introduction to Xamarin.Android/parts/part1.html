Introduction to Xamarin.Android

Duration

10 minutes

Tip: If you are doing this exercise live in a session, make sure to make good use of the instructor, they are online to answer any questions you have!

Goals

This lab walks you through creating a new Xamarin.Android project. There is no code to write here; however, the instructions will point out a few key parts of the project after you have created it.

Required assets

The provided Resources folder for this part contains a subfolder named Completed with a solution you can use to check your work.

Steps

Below are the instructions to implement the exercise. If you would like more detail, please see Xamarin's Android quickstart. It contains a guided walkthrough with screenshots for both Xamarin Studio and Visual Studio that document each step.

Create a new project

  1. Launch your preferred IDE (Xamarin Studio or Visual Studio).
  2. Create a new Xamarin.Android project.
    In Xamarin Studio, use File > New > Solution, then choose the Android App template from the Android > App category.
    In Visual Studio, use File > New > Project, then choose the Blank App (Android) template from the Visual C# > Android category.
  3. Name the app TipCalculator.
  4. Choose a location for the project files.
  5. Use the default values for all other project settings.

Examine the source code (Optional)

  1. Run the app and spend a moment using it so you understand its behavior. The template code typically has a single button that displays the number of times it has been clicked.
  2. Open the file Resource > Layout > Main.axml in the Source view. If you find you are in the Design view, click the Source tab at the bottom of the window.
  3. Examine the XML that declares the Button. In particular, note that the button has an id assigned to it.
  4. Open the file MainActivity.cs.
  5. The template generated an Activity for you. Activities are the core building blocks of Android applications. The Activity attribute for your MainActivity has MainLauncher set to true which makes this Activity your app's primary entry point (i.e. analogous to a Main method from a C# Console Application).
  6. There are two things to note in OnCreate. SetContentView takes an identifier for an XML file (Resource.Layout.Main) as its argument and instantiates the UI. FindViewById takes an id and returns a reference to the view with that id. This is how the code gets access to the views in the UI.

Summary

In this section, you created a new Xamarin.Android application. Subsequent exercises will examine the code in more detail and expand it into a simple but functional app.
